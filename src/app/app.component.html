Angular Core Concepts

Components

Components are the basic building blocks of Angular applications. They control a part of the screen (a view) and are defined by a class that contains application logic and data, and an HTML template that defines the view.

Example: AppComponent is a component.

Modules

Angular applications are modular. They have at least one root module, conventionally named AppModule, which provides the bootstrap mechanism for launching the application. Modules can import functionality from other modules and export their own functionality.

Example: This project uses a standalone component, so a traditional AppModule is not explicitly generated, but the concept of modularity still applies through standalone components and their imports.

Data Binding

Data binding is a mechanism that allows you to synchronize data between the component's class and its template (view).

  • Interpolation (double curly braces): Displays component property values in the template.
  • Property Binding (square brackets): Binds a property of a DOM element to a component property.
  • Event Binding (parentheses): Binds a DOM event to a component method.
  • Two-way Data Binding (banana in a box syntax): Combines property and event binding for input elements (requires FormsModule).

Example: The title above uses interpolation.

Directives

Directives are classes that add behavior to elements in your Angular applications. There are three types:

  • Components: Directives with a template.
  • Structural Directives: Change the DOM layout by adding or removing DOM elements (e.g., *ngIf, *ngFor).
  • Attribute Directives: Change the appearance or behavior of an element, component, or another directive (e.g., ngStyle, ngClass).

Example: We can add a button that shows/hides content using *ngIf.

Services and Dependency Injection

Services are classes that provide specific functionality to components, such as fetching data from a server, logging, or validation. Dependency Injection (DI) is a design pattern used to provide instances of dependencies (services) to a class (component) rather than having the class create them itself.

Routing

Routing allows you to navigate between different views (components) in your application based on the URL. Although this project was initialized without routing, it's a core concept for multi-page applications.
